# HippoAgent — UX R&D Audit & Redesign

> Owner: ui-ux-designer agent · Date: 2026-05-08
> Scope: render HippoAgent UX *sellable* — audit the 3 user surfaces, ship a design system, redesign the highest-value page (`/skills`), tighten the CLI.

---

## 1. Audit — 10 worst UX pain points

Severity legend: `S1` = blocker / first-impression killer · `S2` = friction · `S3` = polish.

| #  | Severity | Surface       | Pain                                                                                   | Impact                                                                                  |
| -- | -------- | ------------- | -------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| 1  | **S1**   | Dashboard     | All HTML is **inline string concatenation** in `dashboard.py` — 2 353 LOC monolith     | Every micro-tweak risks XSS regression; impossible to iterate visually; no design rev.  |
| 2  | **S1**   | Dashboard     | **No design system**. Same accent (`#58a6ff`) for nav links, buttons, KPIs, headlines  | No visual hierarchy — eye doesn't know where to land. "Looks like a 2014 status page."  |
| 3  | **S1**   | Dashboard     | `/skills` is a **dense 7-col table** with 8-char IDs first column                      | Scanning is impossible at >20 skills; "id" column wastes the most valuable real estate. |
| 4  | **S1**   | Dashboard nav | 9 top-level links + emojis (⚡ 🌙 ⚙) collide with the brand                              | Information overload at the first click. No grouping (chat vs admin vs observability).  |
| 5  | **S2**   | Dashboard     | Mixed **emoji + tag** style for status (`🔧 compiled`, `🌀 cf`, `⚡ learned`)             | Color-only signal fails for color-blind users; emoji rendering is OS-dependent.          |
| 6  | **S2**   | Dashboard     | Inline `style="..."` attributes everywhere (`background:#1f6feb;color:#fff;`)         | No reusable components. CSP `style-src 'unsafe-inline'` required.                       |
| 7  | **S2**   | Dashboard     | **No filter / search** on `/skills`, `/episodes`. Only sort = fitness desc.           | At 100+ skills the page becomes a wall of text. CTRL-F is the user's only friend.       |
| 8  | **S2**   | CLI (`code`)  | `/help` lists every command alphabetically — no grouping, no examples                  | Users discover features by accident. Power features (`/repomap`, `/diff`) hidden.       |
| 9  | **S2**   | CLI (`code`)  | Banner shows the same line every prompt, no actionable hint                            | After 50 turns the user has no nudge to consolidate, prune, or promote.                 |
| 10 | **S3**   | TUI / CLI     | Status pane uses `🌙` / `⚡` / `🔧` glyphs in tables — terminal width math broken in 80c | Tables wrap awkwardly on narrow terminals. Glyph-as-key hurts a11y readers.             |

## 2. Design system — tokens & rationale

File: `hippoagent/static/dashboard.css` (single, cacheable, ~18 KB).

### Palette — accessible on dark surface

All combinations clear **WCAG 2.1 AA** on `--c-bg-0` (#0b0e13):

| Token             | Value     | Contrast vs bg-0 | Use                                |
| ----------------- | --------- | ---------------- | ---------------------------------- |
| `--c-text`        | `#e8edf5` | **12.4 : 1**     | Body, page titles                  |
| `--c-muted`       | `#9aa6b8` | **6.0 : 1**      | Secondary text, table headers      |
| `--c-dim`         | `#6b7585` | 3.6 : 1          | Tertiary — large text only         |
| `--c-accent`      | `#6ea8ff` | 6.7 : 1          | Primary CTA, links                 |
| `--c-success`     | `#56d364` | 7.8 : 1          | Promoted status, episode pass      |
| `--c-warning`     | `#e3b341` | 8.7 : 1          | Candidate, mid-fitness             |
| `--c-danger`      | `#ff7b72` | 6.4 : 1          | Failure, retire                    |
| `--c-info`        | `#79c0ff` | 7.7 : 1          | Compiled macro / NREM              |
| `--c-purple`      | `#c8a8ff` | 7.6 : 1          | Counterfactual / REM               |
| `--c-teal`        | `#76d7c4` | 9.0 : 1          | Hebbian / learned                  |

Semantic role aliases (`--c-stage-nrem`, `--c-status-promoted`, …) allow theme swaps without touching components.

### Typography scale — 1.25 ratio

```
--fs-xs   12px   captions / chips / table th
--fs-sm   13px   meta lines, secondary
--fs-base 15px   body
--fs-lg   17px   card / section title
--fs-xl   22px   page title
--fs-2xl  28px   KPI value
--fs-3xl  36px   hero KPI (overview)
```

Two families:
- `--font-sans` system stack (`-apple-system`, `Segoe UI`, `Inter`, …) for prose & UI
- `--font-mono` (`JetBrains Mono`, `SF Mono`, `Cascadia Code`, `Menlo`, …) for **all numeric values, IDs, and code** — keeps numbers aligned in tables and KPIs

### Spacing — 4 px scale, no magic numbers

`--s-1` = 4 px, `--s-2` = 8, `--s-3` = 12, `--s-4` = 16, `--s-5` = 20, `--s-6` = 24, `--s-8` = 32, `--s-10` = 40, `--s-12` = 48, `--s-16` = 64.

### Geometry & motion

Radii: `--radius-sm 4` · `--radius 6` · `--radius-md 8` · `--radius-lg 12`.
Motion: 120 ms / 200 ms easings; **respects `prefers-reduced-motion: reduce`**.

### Components shipped

| Component        | Class root          | States                                         |
| ---------------- | ------------------- | ---------------------------------------------- |
| Button           | `.btn`              | default, primary, success, danger, ghost · sm  |
| Input            | `.input`            | default, focus, disabled                       |
| Card             | `.card`             | default, inset, compact                        |
| KPI tile         | `.kpi`              | accent, success, warning, info                 |
| Chip / tag       | `.chip`             | stage-nrem/rem/compiled/schema · status-* · ico-* |
| Filter pill      | `.filter-pill`      | default, hover, active                         |
| Progress bar     | `.bar`              | success, warning, danger                       |
| Skill card       | `.skill-card`       | hover (lift)                                   |
| Empty state      | `.empty`            | —                                              |
| Table (legacy)   | `.table`            | hover                                          |

Light-mode tokens are pre-baked under `html[data-theme="light"]` — to enable, ship a toggle that sets the attribute (one line of JS, no rebuild).

### Why cards over a table

- **Scannable hierarchy** — name and fitness bar dominate, ID demoted to a small mono-font tag in the header.
- **One row = one skill = one decision.** Promote / Retire actions live on the card, no need to drill in.
- **Status + mechanism chips** are visible at a glance. Color + label (no color-only signal).
- **Filter pills** subset the grid client-side; **search** matches name, trigger, or ID prefix.
- **KPI tiles** at the top answer "what's the state of my library?" in 1 second — the very question the page failed to answer before.

### Accessibility

- `role="search"` on the toolbar; pills are `role="tab"` with `aria-selected`
- Fitness bar is `role="progressbar"` with `aria-valuenow` / `min` / `max`
- Skill name is `<h2>` so screen readers can list-jump
- Buttons have `aria-label` including the skill name (no isolated "Promote" button)
- Skip links and focus-visible outlines (2 px accent, 2 px offset) on every interactive element
- Honors `prefers-reduced-motion` (no transforms / fades)

### What's intentionally *not* in this MVP

- Pagination (deferred — current store top-out ~200 skills)
- Drag-to-reorder (no use case yet)
- Bulk select (would require a new API endpoint)
- Light theme toggle (tokens are ready, no UI yet)

## 4. CLI redesign — `engram code`

### Banner

Before:
```
╭─ ⚡ ENGRAM CODE  C:\path  · model · skills 12 · episodes 7 ─╮
/help for commands · Ctrl-D / /quit to exit · [image: /path] to drop a screenshot
»
```

After:
```
╭─ ENGRAM CODE  C:\path  · claude-haiku-4-5  · skills 12 (3↑, 1 compiled)  · episodes 7 ─╮
/help for commands · Ctrl-D or /quit to exit · [image: /path] inlines a screenshot
tip: 8 new episodes since last consolidation — consider running /sleep
»
```

Changes:
- **Removed brand emoji** (`⚡`) — terminal width is precious; the panel border carries the brand.
- Skill counter now shows `(promoted↑, compiled)` — surfaces the *quality* signal, not just the count.
- New `tip:` line — a single contextual nudge, computed from current memory state. It only appears when the user can act:
  - 0 episodes → "first run — type any task and press Enter; after 3+ episodes try `/sleep`"
  - ≥ 8 episodes since last consolidation → "consider running `/sleep`"
  - ≥ 5 candidate skills → "/skills shows them, /promote <id> if proven"
  - default → "/skills to browse what the agent has learned"
- Logic in `EngramCode._contextual_tip()` — single source of truth, returns `None` when nothing's worth saying.

### `/help` — grouped + per-command detail

Before: alphabetical dump of every `_cmd_*` method.

After: grouped by category, with `/help <cmd>` showing the full docstring as a panel.

```
Memory                                                Workspace
 /sleep    Run a sleep consolidation cycle           /repomap  Print the repo map
 /skills   List active skills, sorted by fitness    /diff     Show git diff
 /promote  Promote a candidate skill                /review   Ask agent to review file
 /retire   Retire (archive) a skill
 /forget   Wipe ALL persistent memory               Model
                                                     /model     Switch executor model
Session                                              /provider  Switch LLM provider
 /status   Show workspace + memory status            /plan      Toggle plan mode
 /clear    Clear the screen
 /help     Show available slash commands
 /quit     Exit the session

/help <command>  shows the full description for one command
```

### New commands shipped

- `/promote <id-prefix>` — mirrors the dashboard action; closes a long-standing CLI/web parity gap.
- `/retire <id-prefix>` — same; asks confirmation by default.

Both use a shared `_resolve_skill_id()` helper so 8-char prefixes resolve like everywhere else in the codebase.

## 6. What I deliberately did NOT do

- **Did not redesign `/`, `/episodes`, `/active-memory`, `/lineage`, `/metrics`, `/events`, `/chat`, `/settings`, `/welcome`** — same template/static rig is now in place; they can be migrated incrementally without disrupting the security-hardened endpoints.
- **Did not touch `ide.py` or `dashboard.py`'s settings & permissions APIs** — Sprint 1 hardened scope.
- **Did not migrate `tui.py`** — Textual has its own theming engine; cross-port would be a separate task.
- **Did not introduce a JS framework, build step, or npm dep** — keeps the dashboard a single `pip install` away from running, matches the codebase ethos.
