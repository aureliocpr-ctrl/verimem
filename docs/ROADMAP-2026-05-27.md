### P1 — Sandbox Bash (2 settimane)

**Obiettivo**: deny-by-default per execute_command capability. Allowlist + dry-run + cwd jail + timeout + env scrub + audit log.

#### P1a — Sandbox engine
- `engram/sandbox.py` con `SandboxedShell` class
- Allowlist regex pattern matching (file:read, git:read, pytest:..., echo, etc)
- Denied operations: rm -rf, format, mkfs, dd, network exec
- Cwd jail: lock execution to whitelisted dirs
- Timeout 60s default
- Env scrub: nasconde secrets/tokens

#### P1b — Integration con tool Bash MCP
- Tutti tool che eseguono shell devono passare attraverso sandbox
- Override `dangerouslyDisableSandbox` solo con user mandate verified

#### P1c — Audit log integrato
- Ogni comando logged in `~/.engram/audit/bash-YYYYMMDD.jsonl`
- Replay possibility

**P1 total**: ~14h work, 2 settimane spread
